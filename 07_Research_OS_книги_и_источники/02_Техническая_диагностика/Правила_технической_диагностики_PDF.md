# Правила технической диагностики PDF

## Зачем нужна диагностика
Перед анализом книги нужно понять, что перед нами:
1. текстовый PDF;
2. скан без текстового слоя;
3. PDF с плохим OCR;
4. PDF с таблицами и схемами;
5. старый DOC/DOCX;
6. поврежденный или проблемный файл.

## Почему Cursor может не видеть книгу
Cursor может не прочитать файл, если:
- PDF является набором изображений;
- нет текстового слоя;
- OCR сделан плохо;
- файл слишком большой;
- файл лежит вне рабочей папки проекта;
- имя файла слишком длинное или содержит сложные символы;
- файл имеет старый формат DOC;
- документ защищен;
- в PDF сложная структура: две колонки, таблицы, схемы, сноски;
- страницы повернуты или имеют плохое качество скана.

## Как исправлять
1. Проверить, выделяется ли текст в PDF вручную.
2. Если текст выделяется — использовать быстрый PDF → Markdown.
3. Если текст не выделяется — сначала сделать OCR.
4. Если файл DOC — конвертировать в DOCX или Markdown.
5. Если файл большой — не анализировать целиком, а сначала извлечь оглавление.
6. Если книга нужна только для одного билета — конвертировать только нужные главы.
7. Если книга базовая для многих билетов — можно конвертировать полностью, но по главам.

## Инструменты
Для текстовых PDF:
- PyMuPDF4LLM;
- Docling;
- Marker;
- pdfplumber;
- pypdf.

Для сканов:
- OCRmyPDF;
- Tesseract OCR;
- Ghostscript.

Для DOC/DOCX:
- Pandoc;
- LibreOffice;
- Microsoft Word export;
- python-docx для DOCX.
