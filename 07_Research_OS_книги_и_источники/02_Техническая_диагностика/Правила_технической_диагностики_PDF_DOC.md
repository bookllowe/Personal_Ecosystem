# Правила технической диагностики PDF, DOC и DOCX

## Зачем нужна диагностика

Перед анализом книги нужно понять, что перед нами:

1. текстовый PDF;
2. скан без текстового слоя;
3. PDF с плохим OCR;
4. PDF с таблицами и схемами;
5. старый DOC;
6. DOCX;
7. поврежденный или проблемный файл.

## Почему Cursor может не видеть книгу

Cursor может не прочитать файл, если:

- PDF является набором изображений;
- нет текстового слоя;
- OCR сделан плохо;
- файл слишком большой;
- файл лежит вне рабочей папки проекта;
- имя файла слишком длинное или содержит сложные символы;
- файл имеет старый формат DOC;
- документ защищён;
- в PDF сложная структура: две колонки, таблицы, схемы, сноски;
- страницы повернуты или имеют плохое качество скана;
- файл находится во внешней папке, а не в открытом workspace Cursor.

## Как работать с внешней папкой

1. Сначала зафиксировать путь к папке.
2. Затем создать реестр файлов.
3. Затем проверить доступность файлов из Cursor.
4. Затем определить формат каждого файла.
5. Затем решить, нужно ли копировать файл в проект.
6. Затем извлечь оглавление.
7. Затем работать только с нужными главами.

## Как исправлять проблемы

1. Если текст выделяется в PDF — использовать PDF → Markdown.
2. Если текст не выделяется — сделать OCR.
3. Если PDF читается плохо — сделать повторный OCR.
4. Если файл DOC — сначала конвертировать в DOCX.
5. Если файл большой — сначала извлечь оглавление.
6. Если книга нужна только для одного билета — конвертировать только нужные главы.
7. Если книга базовая для многих билетов — можно конвертировать полностью, но по главам.

## Инструменты

Для текстовых PDF:

- PyMuPDF4LLM;
- Docling;
- Marker;
- pdfplumber;
- pypdf.

Для сканов:

- OCRmyPDF;
- Tesseract OCR;
- Ghostscript.

Для DOC/DOCX:

- Pandoc;
- LibreOffice;
- Microsoft Word export;
- python-docx для DOCX.
