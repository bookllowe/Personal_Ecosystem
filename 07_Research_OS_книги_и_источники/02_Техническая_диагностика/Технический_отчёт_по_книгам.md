# Технический отчёт по книгам

## 1. Назначение

Этот файл фиксирует техническое состояние книг из внешней папки:

```text
C:\Users\User\Documents\Книги для магистратуры
```

Цель диагностики — определить, какие книги можно сразу конвертировать в Markdown, какие требуют OCR, какие требуют конвертации DOC/DOCX, а какие нужно сначала вручную проверить.

Дата диагностики: 2026-07-22  
Доступ к внешней папке: **получен** (папка видна из среды выполнения).

## 2. Главное правило

На этапе технической диагностики запрещено анализировать содержание книги целиком. Разрешено проверять только:
- доступность файла;
- формат;
- размер;
- количество страниц;
- наличие текстового слоя;
- качество извлечения 1–2 тестовых страниц;
- наличие оглавления;
- потребность в OCR;
- рекомендуемый следующий шаг.

# Метод диагностики

## Проверка 1. Доступность файла

Проверен внешний путь:

```text
C:\Users\User\Documents\Книги для магистратуры
```

Результат: **папка доступна**. Все 6 файлов из реестра найдены.

## Проверка 2. Формат файла

Определены расширения:
- PDF — 5 файлов;
- DOC — 1 файл;
- DOCX — 0 среди проверяемых.

## Проверка 3. Текстовый слой PDF

Для PDF проверено:
- извлечение текста через PyMuPDF (1–2 первые страницы + внутренняя страница);
- наличие закладки/TOC;
- признаки слабо заполненных текстом страниц.

Признаки плохого OCR на пробе **не подтверждены** для файлов 1–4 и 6.

## Проверка 4. DOC/DOCX

Для DOC (Репина):
- файл открывается как существующий;
- прямой текстовый слой недоступен инструментам PDF;
- нужна конвертация DOC → DOCX (Word / LibreOffice), затем Markdown (Pandoc / экспорт).

## Проверка 5. Оглавление

- Автоматические смысловые закладки: почти нигде нет.
- У Данилевского есть технические закладки (`title.pdf`, `t1.pdf` и т.п.) — это **не** рабочее оглавление.
- Оглавление нужно извлекать по первым страницам / вручную позже.

## Проверка 6. Решение по маршруту обработки

### Маршрут А. Текстовый PDF
PDF → Markdown → оглавление → карта глав → тезисы.  
Назначен: Селиванова; Смоленский; Тош; Данилевский (по главам).

### Маршрут B. Скан без текста
Не назначен ни одной книге из текущей шестёрки.

### Маршрут C. Плохой OCR
Не назначен.

### Маршрут D. DOC
DOC → DOCX → Markdown → оглавление → карта глав.  
Назначен: Репина.

### Маршрут E. Сложный учебник со схемами
PDF → оглавление → выбор глав → выборочная конвертация → ручная проверка схем/таблиц.  
Назначен: Короткова/Студеникин.

# Предварительные ожидания по книгам

## Короткова / Студеникин
Ожидаемый тип:
методическое пособие с большим количеством схем и таблиц.

Предполагаемый риск:
таблицы и схемы могут плохо конвертироваться в Markdown.

Фактический диагноз:
текстовый слой есть; OCR не нужен; схемы подтверждены пробой.

Предварительный маршрут:
PDF → диагностика текстового слоя → извлечение оглавления → выбор глав → Markdown по главам (**маршрут E**).

## Селиванова
Ожидаемый тип:
педагогическое пособие по воспитанию.

Предполагаемый риск:
если файл сканированный, потребуется OCR.

Фактический диагноз:
текстовый PDF, OCR не нужен.

Предварительный маршрут:
PDF → оглавление → главы по воспитанию → тезисы для билетов 5–8, 11, 17 (**маршрут A**).

## Смоленский
Ожидаемый тип:
текстовый PDF по теории и методологии истории.

Предполагаемый риск:
возможны сложные методологические термины, но технически файл вероятно читается.

Фактический диагноз:
текст извлекается хорошо.

Предварительный маршрут:
PDF → Markdown по главам → тезисы по методологии (**маршрут A**).

## Тош
Ожидаемый тип:
текстовый PDF по мастерству историка.

Предполагаемый риск:
переводной текст, важна аккуратная терминология.

Фактический диагноз:
текстовый PDF, OCR не нужен.

Предварительный маршрут:
PDF → оглавление → главы о факте, источнике, методе → тезисы (**маршрут A**).

## Репина
Ожидаемый тип:
старый DOC-файл.

Предполагаемый риск:
Cursor может плохо читать старый DOC.

Фактический диагноз:
файл доступен; текст напрямую не извлечён; нужна DOC-конвертация.

Предварительный маршрут:
DOC → DOCX через Word или LibreOffice → Markdown через Pandoc/Docling/ручной экспорт (**маршрут D**).

## Данилевский
Ожидаемый тип:
PDF-курс лекций по Древней Руси.

Предполагаемый риск:
может быть скан или частичный OCR; объём большой.

Фактический диагноз:
текстовый слой есть, OCR не нужен; 398 страниц — работать только по главам.

Предварительный маршрут:
PDF → извлечение оглавления → главы для билетов 31–33 → Markdown по главам (**маршрут A**).
