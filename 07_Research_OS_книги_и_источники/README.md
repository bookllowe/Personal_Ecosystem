Книги, главы, источники, анализ и синтез для лекций и билетов.

## Связь с другими разделами

Нельзя опираться на одну книгу. Результаты идут в Knowledge OS и Exam OS.

## Обязательное содержание билетов перед анализом книг

Перед извлечением оглавлений и созданием карт глав нужно определить, что обязательно должно быть раскрыто в каждом билете.

Это нужно, чтобы анализ книг был целевым:

Билет → обязательные блоки → поиск в книгах → главы → тезисы → ответ.

Основной файл:
`06_Exam_OS_экзамены_и_билеты/Матрица_обязательного_содержания_билетов.md`

## Как экономить токены

- Хранить длинные данные в markdown-файлах.
- Не отправлять в ИИ всю книгу целиком.
- Анализировать документы по главам.
- Использовать краткие карточки.
- Повторно использовать шаблоны.
- Хранить промежуточные результаты.

## Система работы с внешней папкой книг

Оригинальные книги находятся в папке:

```text
C:\Users\User\Documents\Книги для магистратуры
```

Рабочая аналитическая система находится в проекте:

```text
C:\Users\User\Documents\Личная_цифровая_экосистема
```

Работа с книгами строится по этапам:

1. регистрация книги;
2. фиксация пути к оригиналу;
3. техническая диагностика;
4. извлечение оглавления;
5. создание карты глав;
6. связь глав с билетами;
7. выбор нужных глав;
8. конвертация выбранных глав в Markdown;
9. извлечение тезисов;
10. фиксация цитат и страниц;
11. подготовка ответа по билету.

Главное правило:
не анализировать книги целиком и не тратить токены на повторную обработку уже разобранного материала.

Основные файлы:

- `01_Пути_к_оригиналам/Источник_книг.md`
- `00_Реестр_книг/Реестр_книг_для_билетов.md`
- `02_Техническая_диагностика/Правила_технической_диагностики_PDF_DOC.md`
- `04_Карты_глав/Шаблон_карты_главы.md`
- `07_Связь_книг_с_билетами/Матрица_книги_билеты.md`
- `Правила_экономии_токенов_при_разборе_книг.md`

## Этап 2. Техническая диагностика книг

После регистрации книг проводится техническая диагностика.

Результаты диагностики хранятся в:
- `02_Техническая_диагностика/Технический_отчёт_по_книгам.md`
- `02_Техническая_диагностика/Решение_по_маршрутам_обработки.md`
- `02_Техническая_диагностика/Проблемы_доступа_к_книгам.md`

На этом этапе не создаются лекции и ответы по билетам.

## Этап 3. Извлечение оглавлений

На этом этапе извлекаются оглавления приоритетных книг.

Результаты хранятся в:
- `03_Оглавления/Оглавления_приоритетных_книг.md`
- `03_Оглавления/Статус_оглавлений.md`

Оглавления используются для:
- создания карты глав;
- связи глав с билетами;
- выбора глав для Markdown;
- экономии токенов.

## Массовая инвентаризация библиотеки

Источник книг:

```text
C:\Users\User\Documents\Книги для магистратуры
```

В папке находится большой массив файлов для подготовки к магистратуре. Работа с ними ведётся не через чтение всех книг подряд, а через конвейер:

1. полный реестр файлов;
2. статистика библиотеки;
3. первичная классификация;
4. связь с группами билетов;
5. назначение маршрутов обработки;
6. очередь обработки;
7. выбор партий;
8. извлечение оглавлений;
9. карты глав;
10. Markdown только нужных глав;
11. тезисы;
12. билеты.

Основная папка этапа:

`11_Массовая_инвентаризация_281_файл`

Фактическое число файлов на 2026-07-22: **261** (название папки сохраняет ориентир «~281»).

## Этап 5. Нормализация реестра

После массовой инвентаризации проводится нормализация:

1. выбор канонических версий файлов;
2. фиксация дубликатов;
3. исправление ошибок первичной классификации;
4. переразметка “неясной категории”;
5. выделение подпартий 1.1, 2.1 и 3.1.

Основная папка этапа:

`12_Нормализация_реестра`

## Этап 6. Техническая диагностика Партии 1.1

Партия 1.1 — экзаменационный минимум для билетов 1–30.

На этом этапе:
1. проверяется технический статус файлов;
2. фиксируется, что уже было обработано ранее;
3. выделяются нормативные документы;
4. создаётся очередь на извлечение оглавлений;
5. защищаются уже обработанные файлы от повторной обработки.

Основная папка этапа:

`13_Партия_1_1_экзаменационный_минимум`
