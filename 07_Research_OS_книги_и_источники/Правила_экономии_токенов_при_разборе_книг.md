# Правила экономии токенов при разборе книг

## Главное правило

Никогда не отправлять в ИИ всю книгу целиком.

## Почему это важно

Большая книга может содержать сотни страниц. Если отправлять её целиком, модель:

- потеряет структуру;
- смешает важное и второстепенное;
- потратит слишком много токенов;
- может пропустить нужные главы;
- может сделать общий пересказ вместо материала для билета.

## Правильный порядок

1. Сначала путь к книге.
2. Затем метаданные книги.
3. Затем технический статус.
4. Затем оглавление.
5. Затем карта глав.
6. Затем связь глав с билетами.
7. Затем выбор нужных глав.
8. Затем конвертация выбранных глав в Markdown.
9. Затем извлечение тезисов.
10. Затем синтез ответа по билету.

## Что хранить постоянно

- карточку книги;
- оглавление;
- карту глав;
- список ключевых понятий;
- тезисы по главам;
- цитаты со страницами;
- связь с билетами;
- готовые фрагменты ответа.

## Что не отправлять повторно

- полный PDF;
- всю книгу;
- все главы сразу;
- уже обработанные тезисы;
- длинные цитаты без необходимости;
- нерелевантные страницы.

## Какой объём давать ИИ за один раз

- для диагностики: название, оглавление, 1–2 страницы;
- для главы: 5–15 страниц максимум;
- для тезисов: один смысловой раздел;
- для билета: только релевантные тезисы и цитаты;
- для финального ответа: уже сжатые материалы.

## Уровни сжатия

Уровень 1. Полный текст главы.  
Уровень 2. Чистый Markdown главы.  
Уровень 3. Краткий конспект главы.  
Уровень 4. Тезисы для билета.  
Уровень 5. Устный ответ.  
Уровень 6. План ответа на 2 минуты.

## Когда использовать OCR

OCR использовать только если:

- текст не выделяется;
- Cursor не видит текст;
- PDF определяется как изображение;
- важная книга нужна для билета;
- без этой книги нельзя подготовить качественный ответ.

OCR не делать для всех книг сразу.
