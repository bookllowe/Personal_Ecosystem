## 9. Нарушения логики

1. **Файлы не в той папке / размытая граница**
   - Устные клише и формулы есть и в Exam OS, и в Academic Speech OS.
   - Правила экономии токенов — и в Research, и в Cursor OS.
   - Шаблон устного ответа продублирован в `11` и `14`.

2. **Слишком общие файлы**
   - Многие README и «поля для заполнения» повторяют одну схему без содержания.
   - `Типология_источников.md`, `Виды_визуализации.md` — декларативны.

3. **Слишком длинные файлы**
   - `06_…/Матрица_обязательного_содержания_билетов.md` (~2200+ строк) — риск для работы в чате; лучше использовать по фрагментам групп.
   - Лекции 1.1–1.2, `Карта_цифровых_категорий`, `BPMN.md` — длинные (это норма для лекций, но не для навигации).

4. **Пустые шаблоны без применения**
   - Почти все карточки `05`, паспорта `09`, визуализации `12` (кроме BPMN), портфолио `16` — шаблоны без экземпляров.
   - В Research OS пусты папки извлечения: Markdown-главы, тезисы, цитаты, OCR.

5. **Дубли Exam OS ↔ Research OS**
   - Фиксатор источников (Exam) vs матрица книги–билеты (Research).
   - Нужна явная роль: Research = полный учёт; Exam = минимум для ответа.

6. **Дубли Academic Speech OS ↔ Exam OS**
   - Клише и устные формулы пересекаются.
   - Рекомендация: канон формул ответа — Exam; тренировка и диагностика — Speech.

7. **Дубли Project OS ↔ Cursor OS**
   - `Проекты_для_Cursor.md` рядом с промптами Cursor OS.
   - Рекомендация: идеи/паспорта в Project; тексты промптов только в Cursor OS.

8. **Прочее**
   - Статус всех групп билетов «не начато» при уже готовой матрице содержания — логический разрыв статусов.
   - Атлас: карта огромная, но отдельные файлы категорий почти пусты (кроме BPMN).
