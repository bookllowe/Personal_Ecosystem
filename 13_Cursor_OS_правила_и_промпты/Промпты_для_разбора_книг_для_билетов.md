# Промпты для разбора книг для билетов

## Категория: Разбор книг для билетов

### Промпт 1. Создание системы работы с внешней папкой книг

Назначение:
Этот промпт используется, когда книги хранятся отдельно от проекта, во внешней папке, и нужно построить систему их регистрации, диагностики, распределения по билетам и дальнейшего анализа.

Источник книг:
`C:\Users\User\Documents\Книги для магистратуры`

Рабочий проект:
`C:\Users\User\Documents\Личная_цифровая_экосистема`

Главное правило:
Книги не анализируются полностью за один раз. Сначала создаётся реестр, затем техническая диагностика, затем оглавление, затем карта глав, затем связь с билетами, и только потом конвертация выбранных глав в Markdown.

Конвейер:
внешняя папка книг → реестр → диагностика → оглавление → карта глав → связь с билетами → выбор нужных глав → Markdown → тезисы → цитаты → билет.

Что хранится во внешней папке:
- оригинальные PDF / DOC / DOCX.

Что хранится в проекте:
- реестр книг;
- пути к оригиналам;
- техническая диагностика;
- оглавления;
- карты глав;
- связь книг с билетами;
- Markdown-версии выбранных глав;
- тезисы;
- цитаты со страницами;
- пробелы в источниках;
- готовые материалы для билетов.

Ограничения:
- не анализировать все книги целиком;
- не конвертировать все книги в Markdown;
- не делать OCR всех файлов сразу;
- не копировать все PDF в проект без необходимости;
- не создавать HTML и код приложения;
- не переписывать существующую архитектуру проекта.

Связанные файлы:
- `07_Research_OS_книги_и_источники/01_Пути_к_оригиналам/Источник_книг.md`
- `07_Research_OS_книги_и_источники/00_Реестр_книг/Реестр_книг_для_билетов.md`
- `07_Research_OS_книги_и_источники/02_Техническая_диагностика/Правила_технической_диагностики_PDF.md`
- `07_Research_OS_книги_и_источники/04_Карты_глав/Шаблон_карты_главы.md`
- `07_Research_OS_книги_и_источники/07_Связь_книг_с_билетами/Матрица_книги_билеты.md`
- `07_Research_OS_книги_и_источники/Правила_экономии_токенов_при_разборе_книг.md`

### Промпт 1а. Создание системы анализа большого массива книг (внутренняя структура)

Назначение:
Используется вместе с Промптом 1, когда нужно зафиксировать внутренние этапы Research OS после подключения внешней папки книг.

Главное правило:
Книга не анализируется полностью за один раз. Сначала фиксируются метаданные, техническое состояние, оглавление, главы и связь с билетами. Только после этого выбираются нужные главы для конвертации и анализа.

### Промпт 2. Техническая диагностика книг из внешней папки

Назначение:
Провести техническую диагностику книг: определить, какие файлы можно читать как текст, какие требуют OCR, какие требуют DOC → DOCX/Markdown, какие нужно обрабатывать по главам.

Источник оригиналов:
`C:\Users\User\Documents\Книги для магистратуры`

Главное правило:
Не анализировать содержание. Проверять только доступность, формат, размер, страницы, текстовый слой, 1–2 тестовые страницы, оглавление, OCR, следующий шаг.

Результаты:
- `07_Research_OS_книги_и_источники/02_Техническая_диагностика/Технический_отчёт_по_книгам.md`
- `07_Research_OS_книги_и_источники/02_Техническая_диагностика/Решение_по_маршрутам_обработки.md`
- `07_Research_OS_книги_и_источники/02_Техническая_диагностика/Проблемы_доступа_к_книгам.md`

### Промпт 4. Массовая инвентаризация всей папки книг

Назначение:
используется, когда в папке находится большой массив книг и документов. Промпт создаёт полный реестр файлов, статистику, первичную классификацию, связь с группами билетов, маршруты обработки и очередь дальнейшей работы.

Главное ограничение:
содержание книг не анализируется, работа ведётся только по именам, форматам, размерам и путям.

Результаты:
- `07_Research_OS_книги_и_источники/11_Массовая_инвентаризация_281_файл/`

### Промпт 5. Нормализация полного реестра, дубликатов и неясных категорий

Главное ограничение:
содержание книг не анализируется.
