# Личная_цифровая_экосистема

## Назначение проекта

Это единая система обучения, подготовки к экзаменам, работы с книгами, проектирования цифровых инструментов, управления жизнью, развития академической речи и формирования портфолио для комиссии.

## Контекст экзамена

**Направление подготовки:** Педагогическое образование.  
**Направленность:** Историческое образование.  
**Формат подготовки:** работа с билетами, лекциями, литературой, устными ответами, вопросами комиссии и портфолио.

## Главное правило

Сначала структура, потом содержание, потом проект.

## Правило лекций

Лекции должны быть длинными, содержательными и академически сильными. Минимальный объём лекции — 15 минут чтения. Краткий текст допустим только для карточек, конспектов и навигации.

Формат лекций — гибрид:
1. академическая лекция;
2. экзаменационное пособие;
3. материал для запоминания;
4. основа для устного ответа комиссии.

## Правило экзаменационного ответа

Каждый билет должен иметь:
- полную академическую версию;
- среднюю устную версию на 5–7 минут;
- краткую версию для повторения;
- вопросы комиссии;
- академические клише;
- связь с критериями комиссии;
- примеры из преподавания истории.

## Правило запоминания

Материал должен подаваться через:
- логические блоки;
- повторяющуюся структуру;
- тезисы;
- таблицы;
- схемы;
- вопросы;
- примеры;
- речевые формулы;
- связи между понятиями.

## Как пользоваться

1. Новая тема сначала попадает в Атлас категорий.
2. Новые термины фиксируются в Словаре архитектора.
3. Для каждой темы создаётся карточка темы.
4. Для каждого билета создаётся карточка билета.
5. Для каждой книги создаётся карточка книги.
6. Для каждой лекции создаётся карточка лекции.
7. Для каждого проекта создаётся паспорт проекта.
8. Для каждого элемента портфолио фиксируется связь с критериями комиссии.
9. Все большие тексты анализируются по частям.
10. HTML/Web/NLP-панель создаётся только после заполнения примерно 50% содержательной структуры.

## Как экономить токены

- Хранить длинные данные в markdown-файлах.
- Не отправлять в ИИ всю книгу целиком.
- Анализировать документы по главам.
- Использовать краткие карточки.
- Повторно использовать шаблоны.
- Хранить промежуточные результаты.
